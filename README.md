## 📌 Four Pillars of Object-Oriented Programming (OOP)

### 🔹 1. Encapsulation

Encapsulation means **hiding the internal data of a class** and allowing access only through methods (getters and setters).
👉 It helps in protecting data and keeping it safe from unwanted changes.

---

### 🔹 2. Inheritance

Inheritance is a concept where **one class (child) can use the properties and methods of another class (parent)**.
👉 It helps in code reusability and reduces duplication.

---

### 🔹 3. Abstraction

Abstraction means **hiding the implementation details and showing only the necessary features**.
👉 It helps the user focus only on what the object does, not how it does it.

---

### 🔹 4. Polymorphism

Polymorphism means **one method can perform different actions based on the object**.
👉 Same method name, but different behavior.
